# NMPC Solver Failure: Quick Fix Guide

## The Problem
Your NMPC solver is failing with `Maximum_Iterations_Exceeded` and `Infeasible_Problem_Detected` errors repeatedly (~20+ consecutive failures).

## Root Cause (in order of impact):

### 🔴 **#1 CRITICAL: Optimizer Rebuilt Every Iteration**
- **File:** `NMPC_Container.m` line 360
- **Problem:** `buildOptimizerWithObstacles()` is called in every `solve()` call
- **Effect:** Destroys warm-start capability, forces Ipopt to restart from scratch
- **Impact:** 90% of your failures

### 🔴 **#2 CRITICAL: Reference Trajectory Infeasible**
- **File:** `run_mpc_harbour_navigatin.m` line 32
- **Problem:** `x_ref = repmat(x_goal, 1, nmpc.N + 1);` (constant goal)
- **Effect:** Tells solver to stay at (500, 50) while avoiding 166 obstacles = impossible
- **Impact:** Directly triggers Infeasible_Problem_Detected

### 🟠 **#3 MAJOR: Too Many Constraints**
- **File:** `buildOptimizerWithObstacles()` obstacle loop
- **Problem:** All 166 polygon obstacles included every time step
- **Effect:** 1,826 nonlinear constraints in 10-step horizon
- **Impact:** Solver exceeds max iterations

### 🟠 **#4 MAJOR: Warm Restart Disabled**
- **File:** `buildOptimizerWithObstacles()` line ~440
- **Problem:** `opts.ipopt.warm_start_init_point = 'no';`
- **Effect:** Each solve treated as completely new problem
- **Impact:** Prevents using prior solution as initial guess

---

## The Solution (Two Options)

### **Option A: QUICK FIX (try first)**
Modify your existing `NMPC_Container.m`:

**Change 1** - Enable warm starting:
```matlab
% Line ~440 in buildOptimizerWithObstacles()
opts.ipopt.warm_start_init_point = 'yes';  % Change from 'no' to 'yes'
```

**Change 2** - Use dynamic reference:
```matlab
% In run_mpc_harbour_navigatin.m, replace line 32:
% OLD:
% x_ref = repmat(x_goal, 1, nmpc.N + 1);

% NEW:
progress = min(1, t(i) / T_final);  % Smooth progress toward goal
x_ref = zeros(10, nmpc.N + 1);
for k = 0:nmpc.N
    idx = min(i+k, length(t));
    x_ref(4:5, k+1) = x(4:5) + progress * (x_goal(4:5) - x(4:5));
    x_ref(6, k+1) = x_goal(6);
    x_ref(10, k+1) = 70;
end
```

**Change 3** - Limit obstacles:
```matlab
% In solve() before calling buildOptimizer():
% Select only 30 closest/largest obstacles instead of all
obstacles_filtered = selectTopObstacles(obstacles, 30, x0(4:5));
```

## Why These Fixes Work

### Why Persistent Optimizer Works
```
Before: Build → Solve → Build → Solve → Build → Solve (rebuilds waste 70% of time)
After:  Build → Solve → Solve → Solve (structure reused, only parameters change)
```

### Why Dynamic Reference Works
```
Before: "Be at (500,50) forever" + obstacles everywhere = IMPOSSIBLE
After:  "Move gradually toward (500,50)" = FEASIBLE path exists
```

### Why Warm Starting Works
```
Before: Each solve starts from zero, needs 100+ iterations
After:  Each solve starts from prior solution, needs 10-20 iterations
```
